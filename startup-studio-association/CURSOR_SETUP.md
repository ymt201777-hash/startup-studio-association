# Cursor への受け渡し手順書

Cursor でこのプロジェクトを開いて作業するための完全ガイドです。

## 📥 Step 1: Cursor をインストール

1. [Cursor 公式サイト](https://cursor.com) にアクセス
2. 「Download」をクリック
3. あなたの OS（Windows/Mac/Linux）を選択
4. インストール完了後、起動

## 📂 Step 2: プロジェクトを Cursor で開く

### 方法 A: フォルダから開く（推奨）

1. Cursor を起動
2. 左上の「File」→「Open Folder」をクリック
3. このプロジェクトのフォルダを選択：
   ```
   /home/ubuntu/startup-studio-association
   ```
4. 「Select Folder」をクリック

### 方法 B: ターミナルから開く

ターミナルで以下を実行：

```bash
cd /home/ubuntu/startup-studio-association
cursor .
```

## 🔧 Step 3: 依存関係をインストール

Cursor 内のターミナルで以下を実行：

```bash
# Cursor 内でターミナルを開く: Ctrl+` (バッククォート)
pnpm install
```

## 🚀 Step 4: 開発サーバーを起動

```bash
pnpm dev
```

ターミナルに以下のように表示されたら成功：
```
Server running on http://localhost:3000/
```

## 💬 Step 5: Cursor AI を使う（重要！）

Cursor の最大の特徴は **AI アシスタント** です。以下の 2 つの方法で使えます：

### 方法 1: コマンドパレット（推奨）

1. **Ctrl+K**（Mac: Cmd+K）を押す
2. 質問やリクエストを入力
3. Enter キーで実行

**例:**
```
モバイルメニューを実装して。Header.tsx のハンバーガーメニューをクリックしたら
サイドメニューが出てくるようにして
```

### 方法 2: チャットパネル

1. 左サイドバーの「Chat」アイコンをクリック
2. 右側にチャットパネルが開く
3. 質問を入力

## 📋 実装すべきタスク（優先度順）

### 優先度 1: モバイルメニューの実装

**ファイル**: `client/src/components/Header.tsx`

**Cursor への指示:**
```
Header.tsx のハンバーガーメニュー機能を実装してください。

要件:
1. ハンバーガーメニューアイコンをクリックするとサイドメニューが表示される
2. メニューが開いている状態で背景をクリックするか、メニュー内のリンクをクリックするとメニューが閉じる
3. モバイル表示（md以下）でのみハンバーガーメニューを表示
4. スムーズなアニメーション付き
5. メニュー内容は現在のナビゲーションと同じ（ホーム、スタートアップDB、スタジオ一覧、レポート）
```

### 優先度 2: フォント設定の確認

**ファイル**: `client/src/index.css`

**Cursor への指示:**
```
Tailwind CSS で日本語フォント（Noto Sans JP）が正しく適用されているか確認してください。

確認項目:
1. tailwind.config.js で fontFamily が正しく設定されているか
2. index.css で @import されているか
3. 実際のブラウザで日本語が正しく表示されているか
```

### 優先度 3: フィルターのリセットボタン

**ファイル**: `client/src/pages/Startups.tsx`

**Cursor への指示:**
```
Startups.tsx のフィルター機能にリセットボタンを追加してください。

要件:
1. フィルター条件の下に「リセット」ボタンを追加
2. クリックするとすべてのフィルター条件がクリアされる
3. ボタンのスタイルは他のボタンと統一
```

### 優先度 4: ページネーション

**ファイル**: `client/src/pages/Startups.tsx`, `client/src/pages/Studios.tsx`

**Cursor への指示:**
```
スタートアップとスタジオのページにページネーション機能を追加してください。

要件:
1. 1ページあたり 12 件表示
2. ページ番号をクリックして移動可能
3. 「前へ」「次へ」ボタンも実装
4. 現在のページ番号をハイライト表示
```

## 🧪 Step 6: テストを実行

変更後は必ずテストを実行：

```bash
pnpm test
```

すべてのテストが「✓」で表示されたら OK です。

## 🔍 Step 7: ブラウザで確認

1. ブラウザを開く
2. `http://localhost:3000` にアクセス
3. 各ページを確認
4. モバイル表示で確認（F12 → デバイスツールバー）

## 💡 Cursor の便利な機能

### 1. ファイル検索

```
Ctrl+P（Mac: Cmd+P）でファイル名で検索
```

例: `Header.tsx` を探す場合
```
Ctrl+P → "Header" と入力 → Enter
```

### 2. テキスト検索

```
Ctrl+F でファイル内のテキストを検索
Ctrl+H で置換も可能
```

### 3. AI で説明を受ける

コードを選択して **Ctrl+K** で質問：

```
このコードの役割を説明してください
```

### 4. AI でコード生成

```
Ctrl+K → "〇〇の機能を追加して" → Enter
```

## 📝 よくある質問

### Q: Cursor AI を使うのに料金がかかる？

A: 基本的には無料です。ただし、大量に使う場合は有料プランもあります。

### Q: 変更を保存するには？

A: VSCode と同じく **Ctrl+S**（Mac: Cmd+S）で保存されます。

### Q: 間違えて削除した場合は？

A: **Ctrl+Z**（Mac: Cmd+Z）で元に戻せます。

### Q: ターミナルでエラーが出た場合は？

A: Cursor のチャットで以下のように質問：

```
このエラーが出ています: [エラーメッセージをコピペ]
どうやって解決すればいい？
```

## 🎯 作業フロー

1. **Cursor を起動** → プロジェクトを開く
2. **`pnpm dev` を実行** → サーバー起動
3. **ブラウザで確認** → 現在の状態を把握
4. **Cursor AI で指示** → 機能を実装
5. **ブラウザで確認** → 動作確認
6. **`pnpm test` を実行** → テスト実行
7. **変更を保存** → 完了
