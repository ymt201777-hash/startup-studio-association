# Cursor/Nanobanana への受け渡しガイド

このドキュメントは、Manus で実装したスタートアップスタジオ協会サイトを、Cursor（細かい調整）と Nanobanana（デザイン）での作業に引き継ぐための情報をまとめています。

## 📋 現在の実装状況

### ✅ 完了した実装

- **4ページ構成**: トップ、スタートアップDB、スタジオ一覧、レポート
- **Notion API連携**: 完全に実装済み（環境変数設定で有効化可能）
- **フィルター機能**: キーワード、スタジオ、領域、ステージでの絞り込み
- **レスポンシブデザイン**: PC/タブレット/スマートフォン対応
- **コンポーネント設計**: 再利用可能なコンポーネント化済み
- **ホバーアニメーション**: カードの浮き上がり効果実装済み
- **ステージバッジ**: 色分け対応済み
- **テスト**: Vitest による API テスト実装済み

## 💡 引き継ぎ時のポイント

### Cursor へ

1. **ファイルの場所を明確に**: 各ファイルのパスを記載
2. **変更対象を明確に**: どのコンポーネントを修正するか明確にする
3. **テストの実行**: 変更後は必ず `pnpm test` を実行
4. **型チェック**: `pnpm tsc` で型エラーがないか確認

### Nanobanana へ

1. **現在のデザイン仕様を共有**: このドキュメントを参考に
2. **カラーパレットの確認**: 指定されたカラーコードを使用
3. **レスポンシブ対応**: PC/タブレット/モバイルで確認
4. **エクスポート形式**: CSS/Tailwind クラスでの指定
